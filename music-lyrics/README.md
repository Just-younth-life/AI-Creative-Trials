# AI音乐二创工程分层架构
## 整体分层逻辑（与根目录agent工具模块一一对应）
1. 底层支撑层：仓库根目录 agent/
   封装通用工具：歌词等长校验、AI幻觉四层拦截、配器白名单过滤、MIDI节拍匹配工具，全项目统一质量标准。
2. 业务资产层：本music-lyrics文件夹
### 子目录说明
- [lyrics/](./lyrics/)：标准化逐句等长原创歌词文本库，单文件严格对标原曲句式字数；
- [prd/](./prd/)：每首歌曲完整立项PRD，包含市场分析、硬性约束、迭代规划、版权合规条款；
- [docs/](./docs/)：配套执行SOP文档，含编曲配器规则、停顿换气长音演唱规范、AI生成防幻觉完整约束。
## 工程设计收益
1. 单一职责分层，新增歌曲仅需补充lyrics/prd文件，无需改动底层agent工具代码；
2. 统一AI生成校验标准，杜绝旋律漂移、音节错位、配器乱入等幻觉故障；
3. 仓库结构扁平化无嵌套畸形路径，GitHub移动、检索、Blame追溯功能全部正常可用；
4. 开源协作边界清晰，区分歌词创作、PRD需求编写、底层工具优化三类提交。
