# 二、市场背景与行业分析
## 2.1 老龄化社会趋势：银发健康刚需爆发，赛道增量充足
### 1. 行业核心数据
- 人口规模：2025年国内60岁以上老年人口达3.2亿
- 市场体量：银发经济整体市场规模突破9万亿元
- 健康痛点：国内老年人慢性病综合患病率高达78%

### 2. 用户分层痛点拆解
1. 老年用户（使用端）
市面主流健康设备、养老APP操作层级繁琐，学习成本高；独居/空巢老人居家突发疾病、跌倒时缺少完整、快速的应急呼救渠道，慢病缺少长期自动监测工具。
2. 子女用户（监护端）
异地子女无法实时查看长辈健康数据，指标异常无主动预警；老人突发状况只能被动等待电话，无法第一时间介入处置。
3. 社区/医疗机构（服务端）
家庭健康数据孤岛化，缺少标准化居家慢病电子档案；社区医生随访缺少前置数据支撑，慢病干预效率低。

### 3. 市场空白总结
现有产品形态割裂：纯智能硬件便携性差、线上就医工具无日常监测、综合养老平台功能冗余复杂。
行业暂无一体化产品，同时覆盖「低门槛适老操作 + 全周期慢病管理 + 多方联动紧急救援」三大核心需求。

国家持续加码居家智慧养老、慢病数字化管理、居家应急救援三大方向，从硬件补贴、平台建设、线下服务多维度释放产业红利。
本产品「极简适老+健康闭环+熟人急救链」的核心模式完全匹配政策方向，具备社区共建、政企采购、民政项目落地多重渠道优势。

## 2.3 竞品分析：主流适老健康产品多维度对标
选取市场四类代表性产品：老来健康（垂直健康APP）、小度健康屏（智能硬件终端）、悦龄生活（综合养老平台）、微信长辈就医（轻量化工具）横向对比：

| 对比维度 | 老来健康 | 小度健康屏 | 悦龄生活 | 微信长辈就医 | 本产品差异化优势 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| 载体形态 | 独立手机APP | 固定大屏硬件 | 小程序+线下门店 | 微信内置轻工具 | 软硬轻量化结合，无硬件老人也可使用，兼顾居家外出全场景 |
| 操作门槛 | 多层菜单，学习成本高 | 语音交互但功能弹窗繁杂 | 页面信息密集，业务流程冗长 | 仅挂号功能，无健康监测 | 极简适老UI，核心功能两步直达，零认知负担，纯语音操作 |
| 健康闭环能力 | 仅基础数据记录，无干预方案 | 仅采集血压血糖，数据不联动医生 | 侧重线下服务，线上数据断裂 | 无慢病监测模块 | 完整健康闭环：监测→异常预警→线上问诊→用药提醒→健康档案 |
| 应急救援机制 | 仅推送子女消息，无第三方救援链路 | 紧急呼叫仅本机播报，无多人同步 | 依赖线下门店上门，响应慢 | 无急救相关功能 | 熟人急救链：一键同步子女+社区网格员+就近卫生室三方资源 |
| 慢病适配能力 | 慢病分类单一，无长期周期追踪 | 仅基础指标，无周期管理方案 | 主打生活服务，慢病模块薄弱 | 仅就医挂号，无慢病管理 | 针对78%老年慢病群体定制周期监测、指标分析、用药提醒 |
| 场景覆盖 | 仅子女辅助操作，老人难独立使用 | 固定摆放，外出无法携带 | 重度依赖线下门店，居家场景薄弱 | 仅覆盖就医单次场景 | 独居、多代同住、居家、外出全场景适配 |

### 竞品总结
当前行业竞品普遍存在四大短板：功能割裂、操作复杂、急救链路缺失、慢病管理碎片化。
市场缺少一款轻量化、低学习成本、同时覆盖慢病长期管理与多方联动急救的一体化适老健康工具，细分赛道存在明显蓝海空白。

## 2.4 市场机会与产品差异化定位
### 一、四大核心市场机会
1. **人口刚需机会**
3.2亿老年人口+78%慢病患病率，居家长期健康监测、突发急症救援是刚性民生需求，现有产品无法一站式完整满足。
2. **政策红利机会**
多项国家级政策扶持智慧居家养老、适老智能设备、居家应急体系，产品可打通民政、社区、医疗机构渠道，获取项目落地资源与补贴。
3. **用户痛点机会**
数字鸿沟导致老年人无法独立操作复杂健康产品；异地子女监护信息不对称；居家老人突发状况救援链条断裂，均是行业长期未解决的核心痛点。
4. **赛道空白机会**
市面产品要么偏重硬件、要么仅做线上问诊，缺少轻量化、低门槛、慢病管理+多方急救一体化的适老健康工具。

### 二、四大核心差异化定位（区别于全品类竞品）
#### 1. 极简适老：填平数字鸿沟，高龄人群无门槛使用
页面剔除广告、冗余弹窗、多级菜单；采用超大字体、高对比度配色、全语音一键触发；所有核心功能最多两步操作，无需打字、复杂选择，适配低智能手机操作能力的高龄老人。

#### 2. 健康闭环：全周期慢病数字化管理体系
打通「指标监测-异常预警-用药提醒-线上问诊-电子健康档案-社区随访」完整业务链路；自动留存血压、血糖、心率等慢病数据，周期性生成健康报告，异常指标同步推送子女与签约社区医生，解决行业数据碎片化痛点。

#### 3. 零认知负担：轻量化设计，无需学习上手即用
摒弃冗长新手引导、复杂初始配置；10秒完成基础绑定，子女可远程协助设置全部功能，老人无需自主操作配置；界面全部使用通俗口语，规避专业医疗术语，降低理解门槛。

#### 4. 熟人急救链：多层联动极速应急救援体系
区别于竞品仅通知子女的单一模式，搭建三层急救网络：一键呼救同步「直系子女+社区网格员+就近社区卫生服务站」，同步推送老人实时位置、既往慢病病史；支持自定义紧急联系人分组，跌倒、突发疾病自动多渠道推送告警，补齐居家老年应急救援行业短板。

## 三、产品定位与目标
### 3.1 产品定位
本产品定位**居家老年轻量化适老健康管理+应急救援一体化工具**。
当下银发健康赛道同质化严重，多数产品要么硬件重、要么操作复杂、急救链路断裂。本产品以「极简低门槛操作、全周期慢病健康闭环、多方联动熟人急救」三大核心差异化竞争力切入市场，精准匹配深度老龄化民生刚需与国家智慧居家养老政策导向，主打普通居家、独居、空巢老年群体，抢占轻量化居家银发健康细分蓝海赛道。

### 3.2 目标用户画像
#### 画像1：核心使用人 - 独居老人 张建国
- 基础信息：68岁，独居，高血压2级，子女长期异地工作，无智能设备使用经验
- 生理特征：老花眼、手指关节不灵活、记忆力衰退、反应速度慢
- 行为痛点：看不懂小字、多层菜单点错、经常忘记吃药；头晕胸闷时不知道怎么联系子女和社区；不会记录血压，去医院说不清近期指标；害怕复杂APP，存在数字恐惧
- 核心诉求：字大、按钮大、点一下就能记录血压；定时提醒吃药；不舒服一键通知子女，不用手动打字打电话

#### 画像2：监护决策人 - 异地子女 李婷
- 基础信息：32岁，一线城市上班族，父母留居老家，每月仅回家1次
- 行为痛点：无法实时掌握父母血压、服药情况；长辈身体异常只能等老人主动打电话，滞后性强；突发跌倒/急症时无法第一时间同步位置与病史；社区、医院信息不通，求助渠道单一
- 核心诉求：远程查看父母健康数据；指标异常主动推送预警；紧急情况同步社区、急救人员；自动生成健康记录方便线下就医

### 3.3 核心价值主张
**大字号·大按钮·不绕路——让每一位长辈都能自己管好健康。**
- 对老人：消除数字门槛，不用学习就能独立完成测压、吃药提醒、一键求救；
- 对子女：远程监护慢病数据，异常提前预警，搭建多层应急救援链路；
- 对社会：贴合居家适老政策，完善社区老年慢病与应急服务体系。

### 3.4 产品目标与成功量化指标
#### 短期运营目标（上线30天）
1. 日活跃用户DAU：新增注册7日留存≥45%，30日DAU稳步上涨
2. 慢病行为指标：人均每周血压记录频次≥4次；用药提醒完成依从率≥70%
3. 用户体验指标：适老化满意度问卷得分≥4.2/5；用户操作误触报错率＜10%
#### 长期核心成功指标（90天）
1. 7日用户留存≥38%
2. SOS一键求助功能月使用率≥12%
3. 子女绑定关联账号渗透率≥60%
4. 线上预约挂号月转化量持续增长

## 五、功能需求详述
### 5.1 首页看板
#### 功能概述
产品默认首页，聚合今日用药提醒、本周血压速览、SOS悬浮按钮、健康资讯入口四大核心模块，老人无需跳转即可查看当日全部关键信息。
#### 用户故事
作为独居老人，我打开首页就能一眼看到今天要吃的药、最近血压高低，不舒服直接点悬浮求救按钮，不用到处找功能。
#### 交互流程
1. 启动App自动进入首页看板；
2. 顶部展示当日健康摘要；
3. 中部用药卡片横向滚动；
4. 下方血压简易趋势图；
5. 底部固定Tab导航；
6. 右下角永久悬浮SOS脉冲按钮。
#### 适老化要点
- 所有模块标题字号≥20px；
- 用药卡片色块区分正常/逾期未服药；
- 趋势图数字放大显示，弱化复杂坐标轴；
- 无自动轮播弹窗、无自动跳转。
#### 验收标准
1. 冷启动首页加载≤1.2s；
2. 所有可点击区域触控尺寸≥48px；
3. 色盲模式下色彩信息可通过文字区分；
4. 无任何广告弹窗自动弹出。

### 5.2 血压测量录入
#### 单线程录入流程
首页点击「记录血压」→ 弹出数值输入框（收缩压/舒张压/心率）→ 一键保存 → 自动回写首页趋势。
#### 页面结构
1. 大标题：记录今日血压
2. 三组独立大数字输入框，加减按钮放大处理
3. 保存/取消双大按钮
4. 底部小字提示正常血压参考区间
#### 交互规则
- 数字输入支持点击加减，无需弹出键盘；
- 数值超出医学标准自动高亮红色提醒；
- 保存成功弹出大字号Toast提示。
#### 验收标准
1. 输入数值范围合规校验（收缩压60-220，舒张压40-140）；
2. 录入后实时刷新首页图表；
3. 断网状态可本地缓存数据，联网自动同步。

### 5.3 用药提醒
#### 药品卡片结构
单张卡片包含：药品名称、服用时段、剂量、距下次提醒倒计时、「已服用」大按钮。
#### 标记已服用交互逻辑
1. 点击「已服用」按钮，按钮变色打勾；
2. 记录当日服药时间，同步至健康档案；
3. 逾期未服药卡片持续高亮提醒；
4. 支持手动新增/删除药品、自定义每日提醒时间。

### 5.4 预约挂号
#### 科室选择流程
挂号入口→科室大色块分类选择→日期选择→时段确认→提交预约→弹窗提示预约成功。
#### 确认弹窗规范
弹窗全屏宽度，大文字展示医院、科室、时间；底部双按钮「确认预约」「取消」，颜色强区分，防止误触。

### 5.5 健康报告与趋势图表
基于 ECharts 实现7天血压折线图，专门做适老化适配：
1. 折线加粗，节点放大圆点；
2. 坐标轴文字放大，取消细碎网格；
3. 正常/偏高区间用底色区分；
4. 底部文字总结本周血压状态（通俗口语，无专业术语）。

### 5.6 一键求助SOS（核心差异化功能）
#### 完整能力
1. 自动拨打预设紧急联系人电话；
2. 同步推送App消息+短信给子女、社区网格员；
3. 一键调取本机位置共享给全部联系人；
4. 长按3秒触发，防止误触；
5. 可手动选择是否同步拨打120急救电话。
#### 交互规范
全局右下角永久悬浮脉冲红色大按钮，页面切换不消失；触发后全屏警示弹窗，大字号提示正在通知联系人。

### 5.7 健康资讯中心
1. 自测快速入口：血压/心率自查指引；
2. 资讯卡片：大图+大标题，单条资讯仅保留核心短句；
3. 药品摘要跳转：从用药卡片直达对应疾病科普资讯；
4. 禁止长文，内容分段留白充足。

### 5.8 个人中心
1. 基础信息弹窗：身高、年龄、基础病史；
2. 快捷跳转入口：用药管理、紧急联系人设置、适老化设置；
3. 退出登录二次确认弹窗，防止误操作退出；
4. 数据导出、隐私设置统一收纳。

## 六、非功能需求
### 6.1 适老化设计硬性规范
1. 字体：正文最小字号18px，标题≥22px，禁止细体字；
2. 触控：所有可点击区域最小尺寸48×48px，按钮间距≥16px；
3. 对比度：文字与背景对比度≥4.5:1，禁止浅灰小字；
4. 手势：仅保留点击、长按两种基础手势，禁用双击、滑动切换复杂操作；
5. 点击反馈：任意按钮点击必须震动+颜色变化双重反馈。

### 6.2 性能要求
1. 首屏冷启动加载时间 ≤1.5s；
2. 页面切换无卡顿，帧率稳定≥55fps；
3. ECharts图表渲染完成≤300ms；
4. App安装包体积安卓≤35M，iOS≤40M；
5. 内存占用峰值不超过200MB。

### 6.3 兼容性要求
1. 移动端：安卓8.0及以上、iOS12.0及以上手机/平板自适应；
2. 字体兼容：跟随系统放大字体，支持系统大号无障碍模式；
3. 页面布局：横竖屏自动适配，不出现文字截断、按钮挤压。

### 6.4 安全性要求
1. 本地健康数据AES加密存储，明文不落地；
2. 用户手机号、病史、位置信息传输全程脱敏；
3. SOS长按3秒触发，防止口袋误触；
4. 紧急联系人号码仅本地缓存，不上传第三方服务器；
5. 无后台静默获取位置，仅用户主动求助时临时授权。

## 附录C 健康数据医学参考范围
### 血压标准（单位：mmHg）
1. 理想血压：收缩压＜120，舒张压＜80
2. 正常血压：收缩压120~129，舒张压80~84
3. 正常高值：收缩压130~139，舒张压85~89
4. 高血压：收缩压≥140 或 舒张压≥90
5. 重度高血压：收缩压≥180 或 舒张压≥110

### 空腹血糖标准（单位：mmol/L）
1. 正常：3.9~6.1
2. 糖耐量异常：6.1~7.0
3. 糖尿病：≥7.0

### 静息心率标准（单位：次/分钟）
1. 正常成年人：60~100
2. 心动过缓：＜60
3. 心动过速：＞100
